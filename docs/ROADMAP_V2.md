# 노션 기반 견적서 관리 시스템 개발 로드맵

노션을 데이터베이스로 활용하여 견적서를 관리하고, 클라이언트가 웹에서 조회 및 PDF 다운로드할 수 있는 시스템

## 개요

| Phase | 설명 | 상태 |
|-------|------|------|
| Phase 1 | 프로젝트 초기 설정 | 대기 |
| Phase 2 | 공통 모듈 및 컴포넌트 개발 | 대기 |
| Phase 3 | 핵심 기능 구현 | 대기 |
| Phase 4 | 마무리 및 배포 준비 | 대기 |

---

## Phase 1: 프로젝트 초기 설정

> 왜 먼저 하는가: 견고한 기반 없이는 기능 개발이 어렵습니다. 환경 설정과 라우트 골격을 먼저 구축해야 이후 작업이 순조롭습니다.

**목표**
- 개발 환경 구성 및 필요한 패키지 설치
- 라우트 구조 및 파일 골격 생성
- 환경 변수 설정

**완료 기준**
- `app/invoice/[id]/` 라우트 골격이 존재한다
- `.env.local`에 `NOTION_API_KEY`, `NOTION_DATABASE_ID` 변수가 설정되어 있다
- 개발 서버가 정상 실행된다

---

## Phase 2: 공통 모듈 및 컴포넌트 개발

> 왜 먼저 하는가: 모든 기능에서 재사용되는 코드를 먼저 만들어야 중복을 방지하고 일관성을 유지할 수 있습니다.

**목표**
- Notion API 클라이언트 초기화 및 공통 함수 구현
- 공통 타입 정의 (`InvoiceData`, `InvoiceItem`)
- 데이터 조회 및 변환 로직 구현
- 견적서 UI 공통 컴포넌트 구현 (헤더, 항목 테이블, 합계 영역, 상태 배지)

**완료 기준**
- `lib/notion.ts`의 `getInvoiceByPageId()` 함수가 Notion에서 데이터를 정상 조회한다
- 존재하지 않는 페이지 ID 요청 시 `null`을 반환한다
- `InvoiceData` 타입으로 데이터가 올바르게 매핑된다
- 공통 컴포넌트가 mock 데이터로 정상 렌더링된다

---

## Phase 3: 핵심 기능 구현

> 왜 먼저 하는가: 공통 모듈이 완성된 후 핵심 기능을 구현합니다. 견적서 조회와 PDF 다운로드가 이 시스템의 핵심입니다.

**목표**
- 견적서 조회 페이지 UI 구현
- 에러 처리 및 유효성 검증
- PDF 다운로드 기능 구현

**완료 기준**
- 고유 URL로 접근 시 견적서 내용이 웹에서 정확하게 표시된다
- PDF 다운로드 버튼 클릭 시 견적서가 PDF로 다운로드된다
- 존재하지 않는 URL 접근 시 적절한 404 안내가 표시된다

---

## Phase 4: 마무리 및 배포 준비

> 왜 마지막인가: 기능이 완성된 후 품질을 높이고 실제 서비스를 위한 최적화를 진행합니다.

**목표**
- 반응형 디자인 적용 (모바일/태블릿/데스크톱)
- SEO 메타데이터 설정
- 전체 사용자 플로우 E2E 테스트
- Vercel 배포

**완료 기준**
- 모바일/태블릿/데스크톱에서 정상 작동한다
- MVP 성공 기준 5개 항목이 모두 통과된다
- Vercel에 배포되어 실제 URL로 접근 가능하다
