# 노션 기반 견적서 관리 시스템 개발 로드맵

노션을 데이터베이스로 활용하여 견적서를 관리하고, 클라이언트가 웹에서 조회 및 PDF 다운로드할 수 있는 시스템

## 개요

노션 기반 견적서 관리 시스템은 프리랜서/소규모 기업을 위한 견적서 발행 및 공유 도구로 다음 기능을 제공합니다:

- **노션 데이터베이스 연동**: Notion API를 통해 견적서 데이터를 실시간 조회
- **견적서 웹 조회**: 고유 URL로 클라이언트가 견적서를 웹에서 확인
- **PDF 다운로드**: 견적서를 PDF 파일로 변환하여 저장/인쇄 가능

## 개발 워크플로우

1. **작업 계획**

   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - 새로운 작업을 포함하도록 `ROADMAP.md` 업데이트
   - 우선순위 작업은 마지막 완료된 작업 다음에 삽입

2. **작업 생성**

   - 기존 코드베이스를 학습하고 현재 상태를 파악
   - `/tasks` 디렉토리에 새 작업 파일 생성
   - 명명 형식: `XXX-description.md` (예: `001-setup.md`)
   - 고수준 명세서, 관련 파일, 수락 기준, 구현 단계 포함
   - API/비즈니스 로직 작업 시 "## 테스트 체크리스트" 섹션 필수 포함
   - 예시를 위해 `/tasks` 디렉토리의 마지막 완료된 작업 참조

3. **작업 구현**

   - 작업 파일의 명세서를 따름
   - 기능과 기능성 구현
   - API 연동 및 비즈니스 로직 구현 시 Playwright MCP로 테스트 수행 필수
   - 각 단계 후 작업 파일 내 단계 진행 상황 업데이트
   - 각 단계 완료 후 중단하고 추가 지시를 기다림

4. **로드맵 업데이트**

   - 로드맵에서 완료된 작업을 완료로 표시

## 개발 단계

### Phase 1: MVP 핵심 구현

#### Task 001: 프로젝트 환경 설정 및 라우트 골격 구축 - 완료

- ✅ `app/invoice/[id]/page.tsx` 라우트 생성 (스켈레톤 UI)
- ✅ `app/invoice/[id]/loading.tsx` 로딩 상태 구현
- ✅ `app/invoice/[id]/error.tsx` 에러 바운더리 구현
- ✅ `lib/notion.ts` Notion 클라이언트 초기화 및 타입 정의 (`InvoiceData`, `InvoiceItem`)
- - `.env.local`에 `NOTION_API_KEY`, `NOTION_DATABASE_ID` 환경 변수 추가
- - `.env.example`에 환경 변수 템플릿 반영

#### Task 002: Notion API 연동 및 데이터 조회 구현 - 우선순위

- `lib/notion.ts`의 `getInvoiceByPageId()` 함수 구현
- Notion 견적서 페이지 조회 (`notion.pages.retrieve`)
- Notion 항목 데이터베이스 조회 (Relation 기반 항목 목록 가져오기)
- Notion 프로퍼티를 `InvoiceData` 타입으로 매핑/변환
- 존재하지 않는 페이지 ID 접근 시 `null` 반환 처리
- Playwright MCP를 활용한 API 연동 통합 테스트

#### Task 003: 견적서 조회 페이지 UI 구현

- `app/invoice/[id]/page.tsx`에서 `getInvoiceByPageId()` 호출 및 데이터 렌더링
- 견적서 헤더 영역 (견적서 번호, 발행일, 유효기간, 상태 배지)
- 클라이언트 정보 표시 (클라이언트명)
- 견적 항목 테이블 (항목 설명, 수량, 단가, 금액)
- 합계 금액 표시 영역
- PDF 다운로드 버튼 배치 (기능 구현은 Task 005에서)
- shadcn/ui 컴포넌트 활용 (Table, Badge, Card, Button)

#### Task 004: 에러 처리 및 유효성 검증 구현

- 존재하지 않는 견적서 ID 접근 시 `notFound()` 호출 (F011)
- 견적서 전용 404 페이지 (`app/invoice/[id]/not-found.tsx`) 구현
- "견적서를 찾을 수 없습니다" 안내 메시지 및 발행자 문의 가이드
- Notion API 오류 시 에러 바운더리 동작 검증
- 잘못된 형식의 ID 요청 처리

#### Task 005: PDF 다운로드 기능 구현

- PDF 생성 라이브러리 설치 및 설정 (`@react-pdf/renderer` 또는 대안)
- `app/api/invoice/[id]/pdf/route.ts` API 라우트 생성
- 견적서 PDF 템플릿 컴포넌트 구현 (헤더, 항목 테이블, 합계, 푸터)
- 클라이언트 측 다운로드 버튼 동작 연결
- PDF 파일명 규칙 적용 (예: `견적서_INV-001.pdf`)
- Playwright MCP를 활용한 PDF 다운로드 E2E 테스트

#### Task 006: 반응형 레이아웃 및 최종 검증

- 견적서 조회 페이지 반응형 디자인 적용 (모바일/태블릿/데스크톱)
- 모바일에서 항목 테이블 가로 스크롤 또는 카드형 레이아웃 전환
- PDF 다운로드 버튼 모바일 최적화 (고정 하단 버튼 등)
- SEO 메타데이터 설정 (`generateMetadata`에서 견적서 정보 반영)
- Playwright MCP를 활용한 전체 사용자 플로우 E2E 테스트
- MVP 성공 기준 5개 항목 최종 검증
