# 개발 워크플로우

## 플로우 개요

```
PRD.md → ROADMAP.md → Shrimp Task Manager → 구현
```

## 각 문서의 역할

| 문서 | 역할 | 수준 |
|------|------|------|
| `PRD.md` | 무엇을 만드는가 (기능 명세) | 비즈니스 |
| `ROADMAP.md` | 어떤 순서로 만드는가 (Phase 계획) | Phase |
| Shrimp Tasks | 어떻게 만드는가 (실행 단위) | Task |

## ROADMAP.md 작성 기준

Phase별로 아래 4가지만 명시한다:

1. **목표** — 이 단계에서 무엇을 만드는가
2. **완료 기준** — 언제 끝났다고 보는가
3. **순서 이유** — 왜 이 순서인가 (의존성)
4. **기술 제약** — 어떤 라이브러리/방식을 써야 하는가

세부 태스크 분해는 Shrimp가 담당한다.

## Phase 작성 순서 원칙

```
Phase 1: 프로젝트 골격 (환경 설정, 라우트 구조)
Phase 2: 공통 모듈 + 공통 컴포넌트 (재사용 코드 먼저)
Phase 3: 핵심 기능 구현
Phase 4: 마무리 및 배포
```

골격 → 공통 → 기능 → 마무리 순서를 지킨다.
